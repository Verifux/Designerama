Motion

Easing & timing.

Real values pulled from RevealOnScroll.tsx, useMagnetic.ts, and usePointerGlow.ts — this is the entire motion vocabulary of the site, no invented curves.

Scroll-reveal ease: cubic-bezier(0.16, 0.84, 0.24, 1)
Reveal duration: 0.8s default (many sections override to 0.6s), viewport amount: 0.15, fires once
Group stagger: 0.09s between children (RevealGroup)
Accordion expand/collapse: 0.4s, cubic-bezier(0.4, 0, 0.2, 1) — height + opacity via Framer Motion AnimatePresence
Accordion chevron rotate: 0.3s, 180deg on open
Magnetic button pull: strength 0.35, max 12px displacement, mouse-only, disabled under reduced-motion
Pointer glow (cards): --mx / --my custom properties set on pointermove, mouse-only, radial gradient radius 360–520px depending on card
Reduced motion

Every motion primitive checks prefers-reduced-motion or Framer's useReducedMotion() and disables itself accordingly — magnetic pull, pointer glow, ambient dot drift, accordion animation duration all degrade gracefully. This is load-bearing accessibility behaviour, not optional polish — don't strip these checks when touching motion code.

Ambient background.

Two layered background effects, both fixed-position and rendered behind all page content. Neither is decorative-only — both respond to scroll and respect reduced-motion.

AmbientDots (components/shared/AmbientDots.tsx)
  • Canvas-based particle field, clustered organically (not a uniform grid) across 6 loose cluster centres
  • Opacity: 0.4 (Portfolio) / 0.55 (Designerama), driven by --dots-opacity
  • Scroll parallax: particles drift at scrollY × -0.04, smoothed via lerp
  • Zoom effect: scales from 1.0 → 1.35 as the user scrolls through the page (1 + scrollPct × 0.35)
  • Particle colour is read from the live --accent CSS variable at mount — always matches brand accent automatically
BgStripe (components/shared/BgStripe.tsx) — Designerama only
  • Fixed diagonal multi-colour gradient wash, rotated -35deg, positioned top-right
  • Scroll parallax: translateY driven by scrollY × 0.08, lerped for smoothness
  • CSS-only gradient (no image asset) — a rainbow-ish teal → violet → magenta → red sweep at low opacity (0.15–0.22 per stop)
  • Portfolio does not use this effect — it's a Designerama-only atmosphere device
MX / BX / AIX / DX Framework

The triad, plus DX.

Designerama markets Verifux's real framework — this is not marketing copy invented for this site, it's the verified structure read directly from Verifux's own heuristics.js. Three questions, three colours, nine pillars, fifty-four checkpoints, plus a separate DX craft score. Never describe this as "8 pillars, 48 checkpoints" or "UX/DX/MX/AIX" — both were early guesses later corrected.

Responsive Design

Breakpoints & grid.

Standard Tailwind breakpoints, no custom scale — plus one site-specific 700px breakpoint for the idx-row number column.

TokenValueBehaviour
—700pxCustom: idx-row number column 64px → 40px, idx-body padding-left 28px → 16px
sm640pxGrid columns collapse to 1, section padding px-8 → px-6
——No lg/xl custom overrides — site relies on Tailwind's default sm: breakpoint almost exclusively
Rules
  • Container max-width: 1180px (max-w-container).
  • Checkpoint/Work card grids: grid-cols-[72px_1fr] mobile → [100px_1fr_auto] desktop.
  • Heading sizes use clamp() throughout rather than discrete breakpoint overrides — check any heading's clamp range before assuming a fixed px value.
  • Custom cursor and magnetic pull disable automatically under (hover: none), (pointer: coarse) — touch devices get native behaviour, not a broken hover effect.
Implementation

Where the source lives.

This codebase has no Figma/tokens.json pipeline — CSS custom properties in app/globals.css are the single source of truth, consumed directly by Tailwind via tailwind.config.ts. Follow these steps to extend the system correctly.

1

Token source

All colour/radius/opacity tokens live as CSS custom properties in app/globals.css, scoped under :root (fallback), [data-brand="portfolio"], and [data-brand="designerama"]. Never hardcode a hex value in a component — reference the token.

2

Tailwind mapping

tailwind.config.ts maps each token to a Tailwind colour utility (e.g. ink → var(--ink)). Known gap: Tailwind's /NN opacity modifier syntax (e.g. text-ink/75) silently does nothing on these tokens, since they're plain var() references, not the RGB-channel-triplet format the modifier requires. Use color-mix(in srgb, var(--x) NN%, transparent) as an arbitrary value instead.

3

Brand switching at runtime

components/theme/ThemeProvider.tsx sets data-brand on the root element per page. Adding a third brand means adding a new [data-brand="x"] block in globals.css with the same token set as the existing two — nothing else needs to change.

4

Static export & deploy

The site builds via npm run export (STATIC_EXPORT=true next build) for static hosting, with a NEXT_PUBLIC_BASE_PATH env var for subpath test deploys. Full deploy mechanics, including a real basePath gotcha with next/image, are documented in docs/DEPLOYMENT.md — read that before touching build config.

5

Project history & decisions

docs/PROJECT-STATUS.md, docs/DECISIONS.md, and docs/CHANGELOG.md track current state, the reasoning behind non-obvious calls, and a dated history of changes respectively. Check these before re-litigating a settled decision.
